## ⚡ Быстрый старт (для тех, кто не любит читать)

### ▶️ "Запусти меня уже!"
```bash
git clone https://github.com/FUYOH666/VoiceToText-Linux.git
cd VoiceToText-Linux
./install.sh
```

Готово! Теперь жми F9 и говори. Всё остальное настроится само.

### ⚙️ "А если хочу покопаться в настройках..."
```bash
# Переключить на самую точную модель (для русского)
./scripts/switch-model.sh large-v3

# Переключить на быструю (для нетерпеливых)
./scripts/switch-model.sh base

# Сменить язык на английский
./scripts/switch-language.sh en

# Перезагрузить всю систему
./scripts/restart.sh
```

## 🎯 Ключевые фичи (или "Почему это круто, а не очередная шляпа")

### 🛠️ **Решение проблем Linux с хоткеями**
Помнишь, как в Linux хоткеи глючат, конфликтуют и вообще не работают? Мы это починили! F9 работает стабильно, без перезагрузок системы и шаманства с настройками. Наконец-то можно просто нажать клавишу и говорить!

### 🔄 **Умное переключение моделей**
Хочешь максимальную точность для русского? Бери large-v3. Нужно быстро накидать заметки? Переключись на tiny. А если баланс важнее всего - medium твой выбор. Всё меняется одной командой, без перекомпиляции!

### 🇷🇺➡️🇺🇸 **Автоматический перевод**
Говоришь по-русски, а нужно английский текст? Нет проблем! Система сама переведёт твою русскую речь в английский. Идеально для:
- Кодинга (AI-модели лучше понимают английский)
- Документации на английском
- Международных чатов

### 🚀 **CPU-first архитектура**
В отличие от большинства решений, которые жрут всю память видеокарты, мы работаем через CPU. Почему это важно:
- ✅ Не конфликтует с твоими ML-проектами
- ✅ Не грузит GPU память (критично для больших моделей)
- ✅ Стабильная работа на любом железе
- ✅ Экономит электроэнергию

### 🎛️ **Гибкая конфигурация**
Всё настраивается через один `config.yaml`. Хочешь поменять хоткей? Пожалуйста. Нужен другой язык? Легко. Другая модель? Без проблем. Никаких хардкодов в коде!

## ⚙️ Настройка (или "Как сделать систему своей")

Все настройки в файле `config.yaml`. Меняй что хочешь:

```yaml
# Модель распознавания
model: "large-v3"  # tiny, base, small, medium, large-v3

# Язык распознавания
language: "ru"     # ru, en, auto

# Автоматический перевод (ru -> en)
auto_translate: true  # Говори по-русски, получай английский

# CPU-only режим (не трогаем GPU)
cpu_only: true       # Спасибо за понимание!

# Горячая клавиша
hotkey: "F9"        # Или любая другая
```

## 🔄 Перезапуск (или "Когда нужно всё сломать и починить")

### 🚀 Полный сброс системы:
```bash
./scripts/restart.sh
```

### 🔧 Что происходит при перезапуске:
1. ✅ Проверяем, что конфиг в порядке
2. ✅ Убиваем все зависшие процессы Whisper
3. ✅ Чистим временные файлы (мусор никто не любит)
4. ✅ Проверяем, что модель на месте
5. ✅ Запускаем всё заново

### 🚨 Когда бежать к перезапуску:
- Поменял модель в конфиге
- Сменил язык
- Изменил настройки
- Система тупит или глючит
- Просто потому что "может поможет"

## 🛠️ Проблемы и решения (или "Когда техника показывает характер")

### 🚨 **F9 не работает** (самая популярная жалоба)
```bash
# Проверить, что клавиша привязана
gsettings get org.gnome.settings-daemon.plugins.media-keys.custom-keybinding:/org/gnome/settings-daemon/plugins/media-keys/custom-keybindings/custom0/ command

# Волшебная таблетка для GNOME
Alt+F2 → r → Enter
```
**Почему это бывает:** GNOME любит перезагружаться. Ничего личного.

### 🚨 **Модель не найдена**
```bash
# Посмотреть, что у нас есть
./scripts/switch-model.sh --list

# Скачать нужную модель
cd ~/Projects/whisper.cpp
bash ./models/download-ggml-model.sh large-v3
```
**Совет:** Large-v3 весит 3GB, запасись терпением и чаем.

### 🚨 **Текст не на том языке**
```bash
# Проверить текущий язык
./scripts/switch-language.sh --current

# Переключить на русский
./scripts/switch-language.sh ru

# Перезапустить систему
./scripts/restart.sh
```
**Помни:** Для максимальной точности русского нужен large-v3!

### 🚨 **Всё зависло и глючит**
```bash
# Принудительная зачистка
pkill -9 -f arecord
pkill -9 -f "whisper-cli"
pkill -9 -f "whisper-toggle.sh"
rm -rf /tmp/whisper-recordings/
rm -f /tmp/whisper-recording.pid

# Новый старт
./scripts/restart.sh
```
**Профи совет:** Это как Ctrl+Alt+Delete, только для нашего приложения.
